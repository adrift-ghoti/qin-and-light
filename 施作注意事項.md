## A. 專案慣例(所有後續計畫共用)

1. **座標與方向**:`position=0` = 岳山/彈弦端,`position=1` = 龍齦端。SVG 內岳山在右(x 大)、龍齦在左(x 小),所以 x 隨 position **遞減**;position 換算用的弦長起點是岳山木塊內緣 `x=363`(不是弦線線段端點 379),弦長 385 單位。
2. **弦序**:`string=1` 對應 SVG 最上面那條線(`y1=-21.6`),1→7 由上往下。第二步接實體 LED 時以此為準,若發現與實琴視角不符,改映射表、不改資料。
3. **徽位**:`HUI_POSITIONS` 用理論整數比(七徽 = 1/2 弦長),不要從 SVG 圓點座標反推。
4. **徽分記法**:`N.f` = 第 N 徽往第 N+1 徽方向走 f/10,線性內插。整數 `7` ≡ `7.0`。
5. **時間**:`Note.startTime` 是記譜時間;`adsr`/`slide` 的 `t` 相對 startTime,**可為負**(綽/注滑入),渲染不得對負 t 特殊處理。
6. **純邏輯放 `src/model/`,一律先寫測試(TDD)**;UI 元件只讀寫 store,不含業務邏輯。Canvas/SVG/音訊行為以手動跑 dev server 驗證。
7. **`engine` 單例**放 `src/audio/`(如 `src/audio/engineInstance.ts`),不要 export 在元件檔裡,避免 hooks → components 的反向 import。
8. **輸入驗證在 store 層擋,不要只靠 UI**(例:泛音限整數徽位);解析使用者輸入的呼叫要有例外處理與可見回饋,不可 silent fail 或讓例外炸出去。
9. **匯入專案後音檔關聯**:JSON 只存 `fileName`。匯入成功後應提示使用者載入對應音檔,並比對 `durationSec`(誤差 > 0.5s 時警告但不阻擋)。

## B. 已定案的設計決策(2026-07-04 與使用者對齊,亦記錄於高層計畫 §8)

1. **散音視覺 = 整弦齊亮**(不做掃過動畫)。
2. **`PositionStrip` 刻度** = 整數徽 + 每徽細分 10 格(0.1 徽分)。地基切片 Task 9 只畫 13 個整數徽點是刻意簡化,細分刻度在後續計畫補。
3. **七弦定音預設 = 正調** 1=C 2=D 3=F 4=G 5=A 6=c 7=d,存 `meta.tuning`,可改。
4. **同弦音重疊 = 新音切斷前音**:同一條弦新音響起時,前一音光效立即(或經極短淡出,如 50ms)結束;每條弦任一時刻只有一個光點/光帶。實作即時光效(里程碑 6)時依此。
5. **虛音亮度公式**:滑動中亮度 = 該時刻 ADSR 值 × 亮度係數(預設 0.4),拖尾 = 前 0.3 秒軌跡漸淡;**係數與拖尾長度要做成專案可調參數**(展出前現場微調),不要寫死。
6. **徽外按音 = 允許 `13.1`–`13.9`**:龍齦(position=1)視為虛擬「第十四徽」,`13.5` = 十三徽往龍齦內插一半。**注意**:地基計畫 Task 2 的 `parseHuiNotation` 目前寫「`tenth > 0 && hui === 13` 則 throw」——實作時要改為允許,並在 `HUI_POSITIONS` 之外以 1.0 作為第 14 個內插端點;對應測試(`13.5` 應 throw)也要改為驗證內插值。泛音仍限整數徽 1–13。
7. **輸入緩衝可見化 = 地基切片就加狀態列**:工作區固定位置顯示「選中音摘要 + 編輯中:欄位 = 已輸入字串」,不等 NoteInspector。地基計畫目前沒有這個元件,實作時補一個極簡 `StatusBar`(讀 store 的 selectedId/editingField 與輸入 buffer;buffer 需從 `useGlobalShortcuts` 的 ref 改為放進 store 才能被訂閱)。
