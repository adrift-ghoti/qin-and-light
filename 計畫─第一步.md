# 古琴聲光秀 — 第一步「軟體版」實作計畫

## Context / 背景

`計畫.md` 描述了一個三階段專案:用七條 LED 光帶代表古琴七弦,隨音樂展現散音、泛音、按音三種音色與指法。本計畫**聚焦第一步:互動式編輯/播放軟體**——讓使用者匯入一段現成的古琴錄音,一邊播放一邊逐音標記,每個音可設定音色、ADSR 包絡、徽位/按點與走手音軌跡,即時在介面上的古琴看到光帶特效,最後匯出「光帶參數檔」供第二步實體 LED 使用。

關鍵共識(已與使用者確認):
- **平台**:網頁應用(理由見下)。
- **音樂來源**:匯入現成音檔(mp3/wav),非即時合成。
- **光帶映射模型**:每條弦 = 一條 1D 光帶;**光點位置 = 弦上的彈奏點**,**亮度 = ADSR 包絡**,**走手音 = 光點沿弦移動**。
- **目標等級**:認真的展出/裝置 → 需穩定、精確、且資料結構預留實體擴展。
- **減字譜**:第一步純手動標記,但資料結構**預留減字譜欄位**,為第三步 AI(解決同音異位)鋪路。

---

## 1. 平台與技術選型

**推薦:網頁應用(React + TypeScript + Vite)。**

理由:
- **Web Audio API** 提供樣本級精確的時鐘(`AudioContext.currentTime`),這對「音樂時間戳」與「光效同步」是剛需,優於一般 `<audio>` 元素。
- **Canvas / WebGL** 能以 60fps 流暢渲染七弦 + 十三徽 + 光帶特效。
- 跨平台、零安裝;展出時可用 kiosk 模式全螢幕瀏覽器執行。編輯工具與第二步的實體播放本就該解耦——軟體只負責「產生參數檔」。
- 你略懂程式,Web 生態(TypeScript、現成函式庫)學習曲線最平緩、可參考資源最多。

建議技術棧:

| 用途 | 選擇 | 備註 |
|------|------|------|
| 框架 / 建置 | React + TypeScript + Vite | 元件化、型別安全 |
| 音訊播放與時鐘 | Web Audio API(`decodeAudioData` → `AudioBufferSourceNode`) | 精確同步基礎 |
| 波形顯示 | wavesurfer.js,或自繪解碼後的 `AudioBuffer` | 時間軸定位 |
| 工作區位置編輯尺 | Canvas 2D 或簡易 SVG(抽象弦線 + 13 徽記號) | 見 §4 `PositionStrip` |
| 展示區古琴渲染 | 內嵌 `guqin-vector.svg` + Canvas/SVG 疊加光效,如需更炫光效再升級 PixiJS/WebGL | 見 §4 `GuqinDisplay` |
| 狀態管理 | Zustand | 輕量,易做 undo/redo |
| 曲線編輯器(ADSR / 走手音) | 自製 SVG 元件 + 可拖曳控制點 | 見 §4 |
| 存檔 / 匯出 | JSON 匯出入 + localStorage 自動存檔 | 見 §5 |

> 若日後想打包成桌面程式,React 程式可直接用 Electron/Tauri 包裝,不需重寫。

## 2. 核心資料模型

整個專案 = 一個音檔參照 + 一串「音事件(Note)」。建議型別(TypeScript):

```ts
interface Project {
  version: string;
  audio: { fileName: string; durationSec: number; sampleRate: number };
  notes: Note[];
  meta?: { title?: string; tuning?: number[] }; // 預留:七弦定音
}

interface Note {
  id: string;
  startTime: number;          // 音樂中的絕對時間(秒)
  string?: 1|2|3|4|5|6|7;      // 哪一條弦;快捷鍵插入的占位音尚未指定時為 undefined
  type: 'san' | 'fan' | 'an'; // 散音 / 泛音 / 按音
  huiNotation?: string;        // 使用者鍵入的徽分記法原文,如 "7.6"(泛音/按音用,見 §3)
  position?: number;          // 弦上彈奏點,正規化 0..1;按音由 huiNotation 換算而來
  hui?: number;               // 泛音專用:1..13 徽位,由 huiNotation 換算而來
  adsr: EnvelopePoint[];      // ADSR 包絡,可拖曳控制點(見 §4)
  slide?: SlidePoint[];       // 走手音軌跡(僅按音);空陣列=無走手音
  slideTechnique?: string;    // 套用過的指法庫 technique id(見 §3.2),僅供標記/對譜用,實際渲染永遠依 slide 控制點
  jianzipu?: string;          // 預留減字譜欄位(第三步用),第一步可手填或留空
  color?: string;             // 預留:該音光色
}

interface EnvelopePoint { t: number; level: number; } // t:相對音起點秒數(可為負值,見下方設計要點), level:0..1 亮度
interface SlidePoint    { t: number; position: number; } // t:相對秒數(可為負值), position:弦上位置 0..1
```

設計要點:
- `position` 採**正規化 0..1**(0=岳山/彈弦端,1=龍齦端),好處是與弦的實體長度脫鉤,第二步可直接乘以該弦 LED 數量得到像素索引。徽位則另存 `hui`(整數),UI 顯示時換算。
- `adsr` 與 `slide` 都是**控制點陣列**而非固定四參數,這樣曲線編輯器可自由插點;渲染時在點之間做插值(建議 Catmull-Rom 或三次貝茲)。
- `jianzipu` 現在是純字串欄位即可,不需在第一步解析減字譜;它的存在讓第三步 AI 能對齊「音 ↔ 譜字」。
- `string`/`position`/`hui` 皆改為**可選**,以支援 §3.1 快捷鍵「先建占位音、後補細節」的輸入流程:剛插入的音只有 `startTime` 與 `type`,`string` 未指定前視為「未完成」音。
- `Note.startTime` 對應 `adsr`/`slide` 陣列裡的 `t=0`,是**記譜/記錄的時間點**;`huiNotation`/`position`/`hui` 記錄的是該音在 `t=0` 當下的位置(曲線上被記下來的那個錨點),**不代表滑音的起點或終點**——滑音軌跡可以往 `t<0`(如綽/注滑入)延伸,也可以在 `t=0` 之後繼續移動(如接續上/下/吟/猱)。負時間區段的音效與光效仍照 §3 正常規則渲染(不特殊隱藏或消音)。

### 2.1 指法庫資料模型

除了單一 `Note`,系統另外維護一份**指法庫**(technique library),讓使用者能自訂常用指法(例如「上」「下」「綽」「注」「吟」「猱」或自己命名的組合),存成獨立的 JSON 檔匯入/匯出(見 §3.2、§5)。指法分兩種:**音序列型**(套用後新增多個 `Note`)與**走手音形狀型**(套用後修改選中按音的 `slide`)。

```ts
interface TechniqueLibraryFile {
  version: string;
  techniques: TechniqueDefinition[];
}

type TechniqueDefinition = SequenceTechnique | SlideShapeTechnique;

interface SequenceTechnique {
  id: string;                // 唯一識別碼,匯入時若與現有 id 衝突則提示改名或覆蓋
  name: string;               // 顯示名稱,如「連撮」
  type: 'sequence';
  notes: SequenceNoteTemplate[]; // 依序產生的音,offsetTime 皆相對套用當下的播放時間
}

interface SequenceNoteTemplate {
  offsetTime: number;         // 相對套用時間點的秒數偏移(第一個音通常是 0)
  type: 'san' | 'fan' | 'an';
  string?: 1|2|3|4|5|6|7;     // 絕對弦號;未指定則用目前作用弦
  huiNotation?: string;       // 絕對徽分記法;未指定則留白,套用後成占位音待補
}

interface SlideShapeTechnique {
  id: string;
  name: string;                // 如「吟」「猱」「上」「下」「綽」「注」
  type: 'slideShape';
  points: SlideShapePoint[];   // 相對按點的軌跡序列
}

interface SlideShapePoint {
  t: number;                  // 相對按音起點的秒數,可為負值(如「綽/注」的滑入起點發生在記譜時間點之前)
  deltaPosition: number | null; // 相對按點的正規化弦長偏移(單位與 position 相同);null = 套用時提示輸入絕對徽分終點(如「上/下」)
}
```

設計要點:
- `deltaPosition` 用**正規化弦長**(與 `position` 同單位),不是徽分——套用時直接 `note.position + deltaPosition` 就是新的 `SlidePoint.position`,不需查徽分對照表;`null` 的點在套用當下才問使用者要去哪,換算後直接存成該點最終的絕對 `position`。
- 內建的「綽、注、上、下、吟、猱」不再是寫死的特例,而是**app 自帶、開機即載入的一份預設指法庫**,使用者可複製一份修改成自己的版本,和自訂指法用同一套格式與匯入/匯出機制。

## 3. 古琴領域與視覺對應規則

三種音色的視覺語法(此為**設計提案,實作前可再微調**):

- **散音(整弦空弦音)**:無左手按點。視覺上整條弦光帶一起亮起(或從岳山端快速掃過),亮度走 ADSR;音色最飽滿,可用最高基礎亮度/暖色。
- **泛音(在徽位節點輕觸)**:只在 `hui` 指定的徽位節點出現一個清亮、帶微閃爍(bell-like)的光點;古琴僅十三徽可發泛音,UI 應限制只能選徽位。
- **按音(壓弦實音)**:在 `position` 按點出現光點,亮度走 ADSR。若有 `slide`(走手音),光點沿弦移動;移動過程的「虛音」以**較低亮度/拖尾**渲染,與起始「實音」的高亮形成對比。走手音常見指法(綽、注、上、下、吟、猱)都能用 position-over-time 曲線表達。`position` 記的是該音在**記譜時間點(t=0)當下**的位置,不代表滑音起點或終點;滑動可能在記譜時間點之前就開始(t<0,如綽/注滑入),也可能之後繼續(t>0,如接續上/下/吟/猱),渲染時光帶效果照 `t` 值(可負)正常插值顯示。

七弦排列、十三徽位置需依古琴實際比例繪製(徽為泛音節點,七徽在弦中點)。定音預留於 `meta.tuning`。

### 3.1 徽分記法與快捷鍵輸入

為了讓使用者能一邊播放一邊快速用鍵盤標記音(而不必每音都精準滑鼠點擊),採用古琴傳統的**徽分記法**作為按點/徽位的鍵盤輸入格式,並搭配一組全域快捷鍵。

**核心流程**:先用字母鍵瞬間插入一個只帶音色與時間的「占位音」(`string`/`position`/`hui` 皆留空),播放不中斷;再用數字鍵把細節「打」進去。

| 情境 | 按鍵 | 功能 |
|---|---|---|
| 全域 | `Space` | 播放/暫停 |
| 全域 | `←` / `→` | 倒退/快轉(預設步幅約 0.5s) |
| 全域 | `Ctrl+←` / `Ctrl+→` | 倒退/快轉,加大步幅(如 3s) |
| 全域 | `↑` / `↓` | 跳到上一個/下一個已標記的音(依時間排序),並選中 |
| 全域 | `S` | 在目前播放時間插入「散音」占位音,自動選中,弦號留空待填 |
| 全域 | `F` | 插入「泛音」占位音,自動選中,弦號、徽位留空待填 |
| 全域 | `A` | 插入「按音」占位音,自動選中,弦號、按點留空待填 |
| 已選中某音 | `Tab` | 循環切換目前編輯欄位:弦號 → (泛音的徽位 / 按音的按點,散音無此欄位) → 循環回弦號 |
| 已選中某音 | `Shift+←` / `Shift+→` | 微調該音 `startTime`,小步幅(如 10ms) |
| 已選中某音 | `Shift+Ctrl+←` / `Shift+Ctrl+→` | 微調該音 `startTime`,大步幅(如 100ms) |
| 欄位編輯中 | `0`-`9`、`.` | 逐字輸入數值組成徽分記法字串(如按 `7`→`.`→`6` 組成 `"7.6"`);弦號欄位僅接受個位數 1-7 |
| 欄位編輯中 | `Enter` | 確認該欄位數值並換算 |
| 欄位編輯中 | `Esc` | 取消本次輸入,欄位維持原值 |
| 欄位編輯中 | `Backspace` | 刪除最後輸入的一位數字 |
| 已選中某音,未在欄位編輯 | `Delete` / `Backspace` | 刪除該音 |
| 全域 | `Ctrl+Z` / `Ctrl+Y`(或 `Ctrl+Shift+Z`) | Undo / Redo |
| 全域 | `T` | 開啟**指法選單**(見 §3.2):打字搜尋指法名稱,方向鍵選、`Enter` 套用 |

`startTime` 微調與 `Timeline`(§4)原有的滑鼠拖曳調整並存:拖曳適合大範圍調整或配合波形對齊,`Shift+←/→` 適合逐音播放時的逐格精修。

**徽分記法換算規則**:徽分記法 `N.f`(N=徽號 1-13,f=0-9 的十分位)代表「第 N 徽往第 N+1 徽方向走十分之 f」,例如 `7.6` = 第七徽再往第八徽方向走十分之六的位置。換算成正規化 `position` 需要一張 `HUI_POSITIONS[1..13]` 對照表(十三徽在弦上的正規化位置,依古琴實際泛音節點比例排列,並非均分;實作時依實際古琴比例定值),在 `HUI_POSITIONS[N]` 與 `HUI_POSITIONS[N+1]` 之間依 `f/10` 線性內插。整數徽位(如 `7`)等同 `7.0`,直接等於 `HUI_POSITIONS[7]`。泛音只能是整數徽位(UI 限制小數位須為 0);按音可用完整徽分小數。

未指定 `string` 的占位音,`Timeline` 上以醒目樣式標示待補,展示區的 `GuqinDisplay`(見 §4)暫不渲染其光帶特效;插入占位音後 `Tab` 焦點預設停在「弦號」欄位,方便立刻打數字補上。

### 3.2 指法庫:套用方式與內建指法

按音的走手音軌跡(`slide: SlidePoint[]`)可以用 §4 的曲線編輯器手繪,但常見指法純手繪太慢,因此提供 §2.1 定義的**指法庫**與 §3.1 的 `T` 快捷鍵套用:

- **套用流程**:`T` 開啟指法選單 → 打字搜尋名稱 → 方向鍵選、`Enter` 套用。
  - **音序列型**(如「連撮」):以目前播放時間為 `t=0`,依序建立 `notes` 裡定義的每個音;`string`/`huiNotation` 未指定的欄位比照 §3.1 占位音規則留白待補。
  - **走手音形狀型**(如「上」「吟」):套用前必須先選中一個「按音」;套用後用 `points` 產生 `SlidePoint[]` 覆蓋該音的 `slide`(`position = 選中音.position + deltaPosition`),並記錄 `slideTechnique = <technique id>`。若 `points` 中有 `deltaPosition: null` 的點(如「上/下」的終點),套用時會先跳出徽分輸入框,換算後的絕對值直接存成該點的 `position`。

**內建指法庫**(app 自帶,開機即載入,`points` 的 `deltaPosition` 皆為正規化弦長):

| 指法 | 類型 | 軌跡特性 | `points` 概要 |
|---|---|---|---|
| 綽 | slideShape | **滑入型**:從下方滑上來,記譜時間點(t=0)剛好到達本音 | `[{t:-0.15, Δ:+0.05}, {t:0, Δ:0}]` |
| 注 | slideShape | **滑入型**:從上方滑下來,記譜時間點(t=0)剛好到達本音 | `[{t:-0.15, Δ:-0.05}, {t:0, Δ:0}]` |
| 上 | slideShape | 本音響起後明顯上滑到新位置並停留 | `[{t:0, Δ:0}, {t:0.3, Δ:null}]`(終點套用時輸入) |
| 下 | slideShape | 本音響起後明顯下滑到新位置並停留 | `[{t:0, Δ:0}, {t:0.3, Δ:null}]`(終點套用時輸入) |
| 吟 | slideShape | 本音響起後小幅來回顫動,漸緩 | 按點 ±0.01,來回 2–3 次 |
| 猱 | slideShape | 本音響起後較大幅來回顫動,漸緩 | 按點 ±0.03,來回 2–3 次 |

「綽/注」是**滑入型**(`t<0` 開始,滑到 `t=0` 到達記譜位置);「上/下/吟/猱」是本音響起後(`t>=0`)才發生的滑動/顫音。這六個不是寫死的特例,只是預設指法庫裡的資料,使用者可以直接複製一份修改存成自己的版本(見 §5 匯入/匯出)。

**控制點的鍵盤微調**:不論是手繪或套用指法產生的曲線,選中某一個控制點後都可用 §3.1 同一套徽分記法(數字鍵 + `.`)輸入該點精確的 `position`,並用 `Shift+←/→` 微調該點的 `t`,與滑鼠拖曳並存。

## 4. UI 元件分解

介面分為**工作區**(編輯用,操作導向、抽象示意)與**展示區**(呈現用,寫實古琴圖 + 光效)兩個區塊,各元件職責單一、介面清楚:

### 工作區(Workspace)

1. **播放列 `TransportBar`** — 載入音檔、播放/暫停、目前時間、播放速度(慢速便於逐音標記)、波形 + 播放游標。資料來源:Web Audio 時鐘。
2. **位置編輯尺 `PositionStrip`** — 畫**單一抽象弦線 + 十三徽記號**(不區分是哪條實體弦);點擊尺上任一點 → 設定**目前選中音**的 `huiNotation`/`position`/`hui`,是 §3.1 數字鍵輸入徽分記法的滑鼠替代方案;搭配 §3.1 快捷鍵(`S`/`F`/`A` 插入占位音、`Tab` 切欄位、`Shift+←/→` 微調時間)。弦號本身不在這條尺上決定,而是由鍵盤數字鍵或 `NoteInspector` 指定。
3. **時間軸 / 音清單 `Timeline`** — 沿時間顯示所有已標記的音(每弦一軌),可點選、拖曳調整 `startTime`、刪除。
4. **音參數檢視器 `NoteInspector`** — 選中某音後編輯:音色型別、弦、位置/徽位、**ADSR 曲線編輯器**、**走手音軌跡編輯器**、減字譜欄位。

**曲線編輯器(ADSR 與走手音共用一個底層元件)**:SVG 畫布 + 可用滑鼠新增/拖曳/刪除控制點,點間插值即時預覽。ADSR 的 Y 軸是亮度(0..1),走手音的 Y 軸是弦上位置(0..1),X 軸都是相對音起點的時間。走手音的曲線可以純手繪,也可以用 `T` 快捷鍵開指法選單套用 §3.2 的指法庫模板後再微調;選中控制點支援徽分記法數字輸入與 `Shift+←/→` 微調(§3.2)。

### 展示區(Display)

5. **古琴展示 `GuqinDisplay`** — 讀取專案根目錄的 `guqin-vector.svg`(七弦十三徽的寫實向量圖)作為底圖,**即時**讀取播放時鐘,根據 `notes` 在對應弦的路徑上疊加光點/光帶特效(亮起/沿弦移動/消失)。純呈現、**不接受點擊輸入**,只負責把工作區編輯好的資料視覺化。`HUI_POSITIONS`(§3.1)應直接由這份 SVG 內十三個徽記號的座標換算,確保工作區的抽象尺與展示區的寫實圖位置一致。

### 其他

6. **匯出面板 `ExportPanel`** — 匯出/匯入 Project JSON;預留「編譯成 LED 影格」選項(第二步)。另有**指法庫**區塊:匯入指法庫 JSON(`id` 衝突提示改名或覆蓋)、匯出指法庫(可選全部或只匯出自訂部分),見 §2.1、§3.2、§5。

## 5. 匯出格式(銜接第二步)

第一步主要匯出 **Project JSON**(上方 `Project` 結構),它同時是存檔與第二步的輸入。

為第二步預留一個**可選的「編譯」輸出**:把以時間為單位的 notes,取樣成**以 LED 影格為單位**的逐燈亮度序列(例如每秒 30/60 影格,每條弦 N 顆燈)。這層轉換邏輯第一步可先不做,但 JSON 結構要讓它**容易實作**——這正是 `position` 正規化、`adsr`/`slide` 用時間軸控制點的原因。

另外匯出**指法庫 JSON**(§2.1 的 `TechniqueLibraryFile`),與 Project JSON 是獨立檔案:指法庫描述「怎麼產生音/軌跡」,是可重複套用、跨專案共用的模板;Project JSON 存的是套用後產生的具體 `Note`,兩者不混在一起。

## 6. 分階段建置順序(里程碑)

每個里程碑都應可獨立執行與驗證:

1. **骨架 + 音訊**:Vite/React/TS 專案;載入音檔、Web Audio 播放、波形 + 播放游標。
2. **靜態畫面兩件**:工作區 `PositionStrip`(抽象弦線 + 十三徽記號)與展示區 `GuqinDisplay`(內嵌 `guqin-vector.svg`);`HUI_POSITIONS` 由 SVG 徽記號座標換算。
3. **最簡音事件**:按 `S` 在當前時間建立散音占位音;`Timeline` 顯示;可刪除;同步實作 §3.1 的 `F`/`A` 占位音插入(泛音/按音,細節留待里程碑 4)與 `Space`/`←`/`→`/`↑`/`↓` 播放控制快捷鍵。
4. **音色三型 + 位置**:散音/泛音/按音切換;泛音限徽位;按音可選按點;同步實作 `Tab` 欄位切換與徽分記法數字輸入(§3.1),含 `HUI_POSITIONS` 對照表與換算函式。
5. **ADSR 曲線編輯器**:控制點拖曳 + 插值。
6. **即時光帶特效**:展示區 `GuqinDisplay` 隨播放時鐘在 `guqin-vector.svg` 對應弦上渲染亮度/位置,音播完即消失。
7. **走手音軌跡編輯器 + 虛音渲染**:按音光點沿弦移動、虛音拖尾;控制點支援徽分記法/`Shift+←/→` 鍵盤微調。
8. **指法庫**:內建指法庫(綽/注/上/下/吟/猱)預載;`T` 快捷鍵指法選單(打字搜尋、套用);音序列型與走手音形狀型兩種套用邏輯;指法庫 JSON 匯入/匯出(ExportPanel)。
9. **減字譜欄位 + JSON 匯入/匯出 + 自動存檔**。
10. **打磨**:undo/redo(含快捷鍵 `Ctrl+Z`/`Ctrl+Y`)、播放速度、時間量化(quantize)。

第 1–6 個里程碑完成即是一個「可用的原型」;7–10 讓它達到「認真展出」品質。

## 7. 第二步 / 第三步 方向性建議(本計畫不深入)

- **第二步(實體 LED)**:七弦各約 120cm,適合用**可定址 LED 燈條**(WS2812B/SK6812,常見 60 或 144 LED/m)。控制器可用 **ESP32**(內建 WiFi,適合做即時同步)搭配 FastLED/WLED 之類函式庫;七條燈條的資料量需評估是否分多個資料腳位或用電平轉換。第二步的核心工作是把第一步的 Project JSON「編譯」成逐影格逐燈亮度,並與音檔同步播放。**此步需另立計畫並先做硬體採購/打樣評估。**
  - **解析度評估**:60/m(120cm≈72 顆,間距約 16.7mm)在最窄的徽位間隔(如二徽↔三徽,弦長 1/30≈40mm)只剩約 2.4 顆 LED,走手音滑過會頓;144/m(120cm≈173 顆,間距約 6.9mm)同段約 5.8 顆,滑動明顯更平滑。**建議採 144/m。**
  - **次像素混色(sub-pixel blending)**:渲染走手音光點時,不要把位置硬對齊到最近的實體 LED,改用小數位置比例把亮度分給左右兩顆相鄰 LED(FastLED 常見抗鋸齒做法)。這能再多買到一段解析度,讓 144/m 的滑動看起來是真正連續而非一格一格跳。
- **第三步(AI 自動產生參數)**:因古琴同音異位,AI 需同時參考減字譜——這正是第一步預留 `jianzipu` 欄位的原因。先用第一步人工標記累積一批「音檔 + 參數 + 減字譜」資料當訓練/驗證集,再評估方法(音訊事件偵測 + 譜面對齊)。**此步亦需另立計畫。**

## 9. 驗證方式

- **里程碑 1**:能載入一段古琴 mp3,播放/暫停/拖曳,游標與聲音同步(肉眼/耳朵對齊誤差 < ~50ms)。
- **里程碑 3–4**:播放中用鍵盤快捷鍵(`S`/`F`/`A` 插入占位音)能建立正確型別與時間的 Note,`Timeline` 顯示與 JSON 反映正確;`Tab` 切欄位後,用數字鍵輸入徽分記法或在 `PositionStrip` 點擊,兩種方式都能正確設定選中音的 `position`/`hui`,結果一致;泛音被限制只能選整數徽位。
- **里程碑 6–7**:逐音播放時,光點在**正確的弦、正確的位置**亮起,亮度隨 ADSR 變化,按音走手音時光點平滑移動且虛音較暗;音結束後光點消失。
- **里程碑 8**:`T` 開指法選單能搜尋到內建六種指法;套用「吟」「猱」等固定形狀指法直接產生對應曲線;套用「上」「下」時會提示輸入終點徽分,結果正確反映在曲線上;套用「綽」「注」後,曲線編輯器與展示區都能正確顯示/播放**負時間**(記譜時間點之前)的軌跡與亮度,`Timeline` 上該音仍以 `startTime`(記譜時間)為主要定位點;套用音序列型指法(如自訂的連續散音)能依 `offsetTime` 建立多個音,弦號/徽分照 §2.1 規則留白或帶入絕對值;套用後仍可用滑鼠拖曳或徽分記法/`Shift+←/→` 修改個別控制點或音;指法庫匯出 JSON → 重新匯入 → 指法清單一致,`id` 衝突能正確提示。
- **里程碑 9**:Project JSON 匯出 → 重新匯入 → 畫面與資料完全一致(round-trip 無損)。
- 端到端:用一小段真實古琴錄音(含散/泛/按各至少一個音、含一次走手音)逐音標記完成,匯出檔案結構正確且可重現視覺。
